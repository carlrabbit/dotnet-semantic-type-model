# Architecture

## Purpose

Architecture documents describe the structural design of the system.

Architecture documents are authoritative for:
- component structure;
- boundaries and responsibilities;
- dependency rules;
- integration contracts.

Architecture documents are not authoritative for:
- behavioral truth (see docs/SPECS.md);
- implementation decisions (see docs/DECISIONS.md).
